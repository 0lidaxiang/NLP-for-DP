这一次作业是预测PTT文章的推和嘘的数目，是一个用 CNN 预测类型的问题。

Model Description

因为是一个预测问题，所以显而易见不能找 Sigmoid 这种会收敛的函数，Activation 应该用relu 或者 softplus 之类 。loss function 应该用 mae 这种。accuracy 这个 metrics 也不需要了。

model 第一层一定是 embedding layer ，用以转换文字到 matrix 。之后是先用一层 cnn 1d 来提取特征，因为文字的信息量比较小，所以就不用 max pooling 这一层了。用 cnn 1d 的原因是 word ebedding 排列下来可以组合成一个 matrix ,但是和图像不同的是这个 embedding 每个单独的 column 都是可以代表自己的含义，并不是像 image 一般都是和几个 columns 组合起来才能代表一定含义。

接着用一层 flatten 层用来将输入“压平”，即把多维的输入一维化，常用在从卷积层到全连接层的过渡。最后用一层 dense 来输出两维的 vector 表示对 push、down 的数目预测。

    new_corpus = []

How to improve your perfomance

一开始只是训练 50 次，扔上去发现成绩还不错 11.74458。
后来就想增加次数试试看呢，从 100 次、 150 次、 200 次到 250 次，其实发现效果大概在 150 次左右是最好的。

batch_size 越小的话反而 loss 更差，loss 甚至会降低到 13 或 14，一般在 150 左右是能获得比较好的结果。所以一般最好放大到 7200 甚至更高。
